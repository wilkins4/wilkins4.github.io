Hands-on practice written like support documentation.

These notes collect current lab work for Linux fundamentals, networking, Windows support, and security coursework. The goal is to show how I test, document, validate, and improve instead of only listing tools.

Project Hub Networking Lab

Lab Standard

Each public note should be useful to a reviewer who wants to understand the problem, the environment, the exact checks performed, and what the result proved.

  • Objective: What skill or problem is being tested?
  • Environment: OS, tools, services, and constraints.
  • Procedure: Repeatable commands or steps.
  • Validation: How I know the result is correct.
  • Evidence: Sanitized output, screenshot, or summary.
  • Next step: What would make the lab stronger.
