Homelab / Networking

Homelab Infrastructure & Network Upgrade

A practical home lab and network environment for learning support, networking, backups, DNS, Docker services, and secure access without breaking household reliability.

Project Snapshot

This project is about documenting a real environment clearly enough that changes can be planned, tested, explained, and rolled back.

Networking DNS DHCP NAS Docker Backups
Back to lab notes

Problem

A home network is still a production environment for the people who rely on it. Changes need to be useful for learning, but they also need to protect internet access, family devices, storage, and backups.

Solution

I maintain a documented homelab plan that separates inventory, network layout, service notes, backup expectations, and change records. The emphasis is on small improvements that can be tested and reversed if something fails.

Documented Areas

  • Device inventory for routers, access points, switches, storage, computers, and lab devices.
  • Network notes for DNS, DHCP, Wi-Fi, switching, and common connectivity checks.
  • Service notes for NAS storage, Docker-style services, and local utilities.
  • Backup planning with restore-verification expectations.
  • Security considerations for remote access, account access, and exposed services.

Change Workflow

  • Write the problem and desired outcome before changing settings.
  • Record the current state and any settings that may need to be restored.
  • Make one small change at a time.
  • Validate with connectivity, service, and backup checks.
  • Keep rollback steps with the change note.

What This Demonstrates

  • Network documentation and troubleshooting discipline.
  • Practical understanding of DNS, DHCP, Wi-Fi, local services, and backups.
  • Awareness that technical changes affect real users.
  • Clear written handoff notes for future review.
